## 0. 제품 개요 (Product Overview)

**한 줄 정의**: 매일 정해진 시간에 영양제를 빠뜨리지 않고 챙겨먹기 위한 모바일 친화적 PWA 체크리스트.

**대상 사용자**: 본인(소유자) — 시간대별·식사 전후 흡수 조건이 다른 영양제 6종 이상을 매일 복용해야 하는 사람.

**핵심 가치**:
1. 시각적으로 시간대 흐름이 보이는 스케줄 (5개 블록)
2. 흡수 방해 조합(철분+커피, 철분+마그네슘 등) 안내로 복용 실수 방지
3. 모바일 홈 화면에서 즉시 열어 한 손으로 체크
4. 자정이 지나면 자동 초기화 — 매일 새로 시작

**비-목표 (Non-goals)**:
- 다중 사용자 / 계정 시스템
- 서버·백엔드 (전부 클라이언트 사이드)
- 영양 분석·추천 알고리즘 (단순 체크리스트만)

## 2. 진행 중 / 예정 작업

### 🔴 P0 — 구조 리팩터링 (Robot Architecture 도입)

`#refactor` — [PRINCIPLE.md](PRINCIPLE.md)의 핵심 철학을 코드에 반영하기 위한 분리 작업. 이후의 모든 기능 추가는 이 구조 위에서 이루어진다.

- [ ] **데이터 분리**: 영양제 정의를 `src/data/supplements.js`로 추출 (HTML에서 카드를 동적으로 생성할 수 있도록)
  - 데이터 스키마: `{ id, name, dose, tip, icon, tag, tagStyle, timeBlock }`
  - 시간 블록 정의 분리: `{ id, label, badge, dotColor }`
- [ ] **Storage 모듈 분리**: localStorage 접근을 `src/storage/supplement-storage.js`로 캡슐화
  - 공개 API: `save(state)`, `load()`, `resetIfNewDay()`, `clear()`
  - localStorage 키 이름·날짜 비교 로직을 외부에 노출하지 않음
- [ ] **UI 렌더러 분리**: `src/ui/schedule-renderer.js` — 데이터 객체를 받아 DOM 생성
- [ ] **컨트롤러 분리**: `src/controller/check-controller.js` — 이벤트 위임으로 인라인 onclick 제거
- [ ] **부트스트랩**: `src/app.js`에서 모듈을 조립
- [ ] **CSS 분리**: 인라인 `<style>`을 `styles/main.css`로 추출
- [ ] **localStorage 키를 인덱스 → ID 기반으로 변경** (카드 순서 변경 시에도 상태 유지)
- [ ] **인라인 onclick 속성 제거** ([PRINCIPLE.md](PRINCIPLE.md) §5 안티패턴)
- [ ] [STRUCTURE.md](STRUCTURE.md) §3-4 목표 구조에 맞춰 §1, §3 갱신

### 🟡 P1 — 기능 추가

- [ ] `#feature` **카테고리 탭** — 화면 상단에 탭 네비게이션 추가, 영양제와 운동 보충제를 분리해 관리
  - 탭 1: **영양제** (현재 콘텐츠 — 매스틱검·유산균·종합비타민·오메가3·철분·마그네슘 등)
  - 탭 2: **헬스 보충제** (운동 전후 섭취 — 단백질·크레아틴·BCAA·EAA·프리워크아웃·글루타민 등)
  - 각 탭은 자신만의 시간 블록·카드·진행률을 가짐 (탭 전환 시 상태가 섞이지 않음)
  - localStorage 키를 카테고리별로 스코핑 (`supplementChecks` → `checks:nutrition`, `checks:fitness` 등)
  - 진행률 도트 개수와 "오늘 N/M 복용"의 M 값은 활성 탭에 따라 동적으로 변경
  - 데이터 정의가 두 배가 되므로 [P0 리팩터링](#0-p0--구조-리팩터링-robot-architecture-도입)의 데이터 분리 작업이 선행되어야 깔끔함
  - 탭 디자인: 헤더 바로 아래, 클레이 underline + Noto Serif KR 11~12px (Garden Notes 톤 유지)
- [ ] `#feature` 영양제 추가/삭제/편집 UI (현재는 코드 수정 필요) — P0 리팩터링 선행 필수
- [ ] `#feature` 복용 시간 알림 (Web Push 또는 Notification API) — 푸시 권한·플랫폼 지원 필요, [#discuss](#3-토의-사항) 항목 참조
- [ ] `#feature` **캘린더 + "잘했어요" 도장** — 현재 헤더의 `001일째` 카운터를 월간 달력 그리드로 교체. 그날의 영양제를 모두 복용한 날에는 도장(스탬프) 마크가 찍히는 느낌. 빠뜨린 날은 비어있고, 부분 복용일은 옅은 표시. 시각적으로 "꾸준함"이 한눈에 보이도록.
  - 데이터: 매일의 완수 여부를 `localStorage`에 일자별로 누적 저장 (현재 `supplementChecks`는 오늘 것만 — 자정 자동 초기화 직전에 "오늘 완수 여부"를 이력에 기록하는 훅 필요)
  - UI: 헤더 영역 또는 별도 토글 페이지 (월간 그리드 7×5)
  - 도장 디자인: 클레이 톤 원형 스탬프 + "잘했어요" 또는 ✓ 또는 작은 botanical 마크
  - 부분 완수: 옅은 도트 또는 진행률 원호로 표시
  - 데이터량이 늘어나면 [#discuss 데이터 영속화 범위](#3-토의-사항) 결론에 따라 IndexedDB로 이전 검토
- [ ] `#feature` 7일/30일 복용 이력 시각화 (간단한 히트맵) — 위 캘린더와 별개의 압축 뷰. 캘린더가 우선
